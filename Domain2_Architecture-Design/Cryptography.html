Cryptographic Concepts

A cryptographic algorithm is a set of mathematical functions which uses a (often secret) string of data known as a key, to encrypt/decrypt other data.

    Two main types of encryption algorithms
  • Symmetric Encryption
  • Asymmetric Encryption

  • Hashing algorithms are computed one way - meaning the output cannot be returned to the original plaintext.
    Data States:
  • In-Transit: any data sent across the network
  • At-Rest: Any data currently located in memory, including databases and backups
  • In-Use: Any data actively used by a process

IPSec Framework

    Protocols
  • {{AH}} filters traffic
    • Authenticates the origin of IP packets and guarantees the integrity of the data
    • Does not provide confidentiality as the packet's data section is not encrypted
  • {{ESP}} encrypts
    • Can be used in combination with AH
    • Encrypts a packet to secure the data
    • Adds an ESP header, trailer and integrity check value to the original packet
    Modes
  • Tunnel
    • Encrypts the entire packet and appends a new IP header
    • Used to send communications on non-secure networks
  • Transport
    • Routable
    • Encrypts only the packet data and not the packet's IP headers
    • Generally used on private networks for E2E communications
