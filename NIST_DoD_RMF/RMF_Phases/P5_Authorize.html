Phase 5: Authorize

Authorization of the System

    Authorization Ojective
  • Obtain Authority to Operate approval for the system
    Authorization Tasks:
  1. POA&M Development
  2. SEcurity authorization package
  3. Risk determination
  4. Risk acceptance
  • Information System Owner is primary role for tasks 1 & 2
  • Authorization Official is primary role for tasks 3 & 4
    5 key areas for authorization:
  1. Develop Plan of Action and Milestones (POA&M)
  2. Assemble security authorization package
  3. Determine risk
  4. Determine the acceptability of risk
  5. Obtain security authorization decision

The Accreditation Package

    Should contain:
  • System Security Plan (SSP)
  • Security Assessment Report (SAR)
  • Plan of Action and Milestones (POA&M)
  • May also include additional artifacts needed to obtain the authorization decision
  • Terms & Conditions are required when granting "Interim Operating Authorization"

POA&M Layout

    Identifies:
  • Weakness
    • Non-compliance (N/C)
    • Not applicable (N/A)
    • Inherited
    • Resources required

      • Mitigation identifies how you will reduce the risk of a vulnerability to an acceptable level for operation

  • POC Resources
  • Required Resources
  • Scheduled Completion Dates
  • Milestones with completion dates
  • Changes to milestones
  • Source of weakness
  • Status

Authorization Documents

  • NIST SP 800-37
  • NIST SP 800-39
  • Authorization Package:
    • Executive summary
    • SSP
    • SAR
    • POAM
  • Authorization decision document
      • 4 Types of RMF Authorization
    1. Authorization to Operate (ATO)
    2. Common Control Authorization (CCA)
    3. Authorization to Use (ATU)
    4. Denial of Authorization to Operate (DATO)
      DoD-related Authorization Types
    1. Interim Authorization to Operate (IATO)
    2. Interim Authorization to Test (IATT)

Determine acceptability of risk

  • Explicit acceptance of risk is the responsibility of the Authorizing Official

    • Cannot be delegated

  • Factors for consideration include:
    • Mission and operations needs
    • Security requirements
    • Reputation & unit/agency image
    • Cost
    • Risk guidance

Reciprocity agreements

  • AO Level documents in which AO's on each side of an interface agree to accept the corresponding organization's authorization decision, POA&M's and security posture
  • Roles:
    • Authorizing official (AO)
    • Authorizing official designated representative (AODR)
    • Common control provider (CCP)
    • Chief Information Officer (CIO)
    • Senior information security officer (SISO)
    • Information owner/steward (IO/S)
    Kinds of authorizations:
  • System Authorizations

    • For major application or general support systems

  • Site Authorizations

    • Evaluates the applications and systems at a specific, self-contained location

  • Type Authorizations

    • Evaluates an application or system that is distributed to a number of different locations

    Reauthorizing actions
  • Time driven
  • Event driven
